# Динамическое освещение небосвода

Проект создан как сокращенная версия [проекта глобального динамического освещения](https://github.com/EkatherinaS/itmo-global-dynamic-illumination-2026) для демонстрации изменения освещения небосвода в течение дня.

Для визуализации небесной полусферы использовалась модель распределения яркости **"All Sky Model"** [1]. Она заключается в вычислении яркости для каждой точки небосвода в зависимости от ясности неба, положения солнца и положения самой точки на небосводе.

Авторы статьи использовали стандартные модели распределения яркости CIE Standard Clear и Overcast Sky и на их основе предложили собственную модель, которую назвали **"All Sky Model"**. Дополнительные данные были получены с помощью измерений глобальной освещенности неба над Токио. На основании этих данных был проведен регрессионный анализ, в ходе которого был получен параметр **𝑁𝑒𝑣𝑔** (normalized global illuminance), который принимает значение около **1** при чистом небе и около **0.2** при пасмурных погодных условиях.

Далее было разработано базовое уравнение для модели, описывающее яркость элемента неба **𝐿𝑣𝑎** как функцию от яркости зенита **𝐿𝑣𝑧**, высоты элемента неба **𝛾** и углового расстояния между этим элементом и солнцем **𝜁**.

```
𝐿𝑣𝑎(𝛾𝑠, 𝛾, 𝜁)   =   𝜙(𝛾) ⋅ 𝑓(𝜁)
────────────────     ──────────────
      𝐿𝑣𝑧             𝜙(𝜋/2) ⋅ 𝑓(𝜋/2 − 𝛾𝑠)
```

Так как требовалось покрыть все состояния неба, функции **𝜙(𝛾)** и **𝑓(𝜁)** варьируются в зависимости от четырех переменных коэффициентов **𝑎**, **𝑏**, **𝑐** и **𝑑**, которые, в свою очередь, напрямую зависят от параметра **𝑁𝑒𝑣𝑔**.

```
𝑓(𝜁) = 1 + 𝑏 ⋅ (exp(𝑐 ⋅ 𝜁) − exp(𝑐 ⋅ 𝜋/2)) + 𝑑 ⋅ cos² 𝜁
𝜙(𝛾) = 1 + 𝑎 ⋅ (1 − sin⁰·⁶ 𝛾)
```

В текущей работе данная модель использовалась при отрисовке небосвода: для каждой точки небосвода по ее локальной позиции вычисляются углы **𝜁** и **𝛾**, далее параметр **𝑁𝑒𝑣𝑔** и расположение солнца задаются слайдерами. Это позволяет протестировать небосвод при всех возможных расположениях солнца и ясности неба. Полученные значения яркости записываются в промежуточную **cubemap-текстуру**, по которой впоследствии вычисляется текстура для **irradiance**.

## Ambient-освещение сцены

Далее по полученному распределению яркости небосвода была построена **irradiance-cubemap**, чтобы впоследствии использовать ее как добавку к основному освещению сцены. Подсчет irradiance на основе исходной текстуры окружения выполняется на основании уравнения **rendering equation**:

```
𝐿𝑜(𝑝, 𝜔𝑜) = ∫Ω BRDF ⋅ 𝐿𝑖(𝑝, 𝜔𝑖) ⋅ 𝑛 ⋅ 𝑑𝜔𝑖
```

где:

- **𝐿𝑜(𝑝, 𝜔𝑜)** — энергетическая яркость (radiance), исходящая от точки **𝑝** в направлении **𝜔𝑜**;
- **BRDF** — двунаправленная функция отражательной способности, характеризующая отражающие свойства поверхности;
- **𝐿𝑖(𝑝, 𝜔𝑖)** — энергетическая яркость, падающая на точку **𝑝** из направления **𝜔𝑖**;
- **𝑛** — нормаль поверхности в точке **𝑝**;
- **Ω** — полусфера над поверхностью.

То есть, для ее построения необходимо определить энергетическую яркость **𝐿𝑜** для каждой точки поверхности **𝑝** в направлении **𝜔𝑜**. На практике данная величина вычисляется как сумма произведений значений яркости неба **𝐿𝑖**, полученных на предыдущем этапе, на косинус угла между нормалью к поверхности **𝑛** и направлением на соответствующий участок небосвода **𝜔𝑖**.

Полученные значения записываются во временную **cubemap-текстуру**.

Для использования этих данных в качестве ambient-освещения достаточно выполнить выборку из текстуры по направлению, соответствующему нормали в текущей точке.
